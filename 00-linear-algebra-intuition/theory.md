# 线性代数直觉（零假设版）

> 本文档不假设你学过线性代数。如果你曾经在大学线代课上"听不懂但考过了"，这份文档是为你写的。

## 符号速查表（遇到不认识的符号回来查）

| 符号 | 读法 | 含义 |
|------|------|------|
| `x ∈ R` | x 属于 R | x 是一个实数 |
| `x ∈ R^n` | x 属于 R 的 n 次方 | x 是 n 个实数组成的向量 |
| `A ∈ R^(m×n)` | A 属于 R 的 m×n | A 是一个 m 行 n 列的矩阵 |
| `x · y` 或 `⟨x, y⟩` | x 点 y | 点积（两个向量相乘得到一个数） |
| `A @ x` 或 `Ax` | A 乘 x | 矩阵乘向量 |
| `A^T` | A 转置 | 行列交换 |
| `\|\|x\|\|` | x 的范数 | 向量的"长度" |
| `x^T y` | — | 等价于点积 `x · y` |
| `∑` | 求和符号 | 把后面一串数加起来 |

## §1 向量是什么

### 先说结论

**向量有两个等价的身份**：
1. 几何身份：一个"带方向的箭头"，从原点出发
2. 代数身份：一列数字

这两个身份在线性代数里**完全等价**。学会在两者之间自由切换，线代就成功了一半。

### 几何身份

想象一张纸，纸上画了 x 轴和 y 轴。从原点 (0,0) 画一个箭头到 (3, 2)，这个箭头就是一个二维向量。

```
y
|
2 - - - *  ← 箭头终点 (3, 2)
|      ↗
1 -   ↗
|   ↗
|  ↗
+--------- x
0  1  2  3
```

向量的两个关键属性：**方向** 和 **长度**。起点在哪里不重要（可以平移），重要的是"指向哪、有多长"。

### 代数身份

同一个向量，写成数字就是：

```
    ┌   ┐
v = │ 3 │
    │ 2 │
    └   ┘
```

或者横着写 `v = [3, 2]`（在代码里更常见）。

### 为什么 ML 里到处都是向量

因为**数据天然是向量**。

- 一个用户：`[年龄=25, 身高=170, 体重=65]` → 三维向量
- 一张 28×28 的黑白图片：784 个像素值 → 784 维向量
- 一句话经过词嵌入：比如变成 768 维向量

ML 的核心操作之一就是"在高维空间里找到向量的规律"。

### 向量加法：首尾相接

几何上：把两个箭头首尾相接，从起点到终点画一条新箭头。

```
v1 = [3, 1]    v2 = [1, 2]    v1 + v2 = [4, 3]

   ↗ v2
  ↗  \
 ↗    ↘
------→  v1
```

代数上：对应位置相加。

```
[3, 1] + [1, 2] = [3+1, 1+2] = [4, 3]
```

**这两种理解必须同时在脑子里激活**，而不是二选一。

### 数乘：缩放箭头

`2 * [3, 1] = [6, 2]` — 箭头方向不变，长度变成 2 倍。

`-1 * [3, 1] = [-3, -1]` — 箭头完全反向。

`0.5 * [3, 1] = [1.5, 0.5]` — 箭头缩短一半。

## §2 线性组合与张成

### 线性组合是什么

一个花哨的词，其实就是：**把几个向量，各乘一个数，再加起来**。

```
2 * [1, 0] + 3 * [0, 1] = [2, 3]
```

这里 `[2, 3]` 就是 `[1, 0]` 和 `[0, 1]` 的线性组合，系数分别是 2 和 3。

### 张成（Span）

**问题**：给你两个向量 `v1` 和 `v2`，你可以通过"不同的系数组合它们"到达平面上哪些点？

**答案**：大多数情况下，**可以到达整个二维平面**。

```
任何点 (x, y) = a * v1 + b * v2   （选择合适的 a, b）
```

这个"所有能到达的点的集合"就叫**张成（Span）**。

### 例外：两个向量同方向

如果 `v1 = [1, 0]`、`v2 = [2, 0]`，它们在同一条线上。

无论你怎么组合，都**永远出不了 x 轴这条线**。这时它们的张成是"一条直线"，不是整个平面。

### 这有什么用

ML 里"数据的特征之间互相冗余"（相关性太高）时，相当于这些特征向量"方向接近"，张成的空间小，模型能学到的东西就少。这是**主成分分析 PCA** 的直觉来源。

## §3 矩阵是变换 ⭐ 最重要的一章

### 颠覆性的新视角

大学课本教你"矩阵是一堆数排成方阵"。忘了它。

**正确的视角：矩阵是一个"变形操作"**，它把整个空间（所有向量）一起拉伸、旋转、压扁。

### 一个具体例子

有一个 2×2 矩阵：

```
    ┌     ┐
A = │ 2 0 │
    │ 0 3 │
    └     ┘
```

它代表的变换是：**把空间在 x 方向拉 2 倍，y 方向拉 3 倍**。

验证一下：把向量 `[1, 1]` 喂给它（矩阵乘向量）：

```
A @ [1, 1] = [2*1 + 0*1, 0*1 + 3*1] = [2, 3]
```

原来的 `[1, 1]` 变成了 `[2, 3]`。正好 x 方向变 2 倍、y 方向变 3 倍，和我们说的变换一致。

### 矩阵乘向量的计算规则（现在才讲公式）

```
┌      ┐   ┌   ┐   ┌                  ┐
│ a  b │ @ │ x │ = │ a*x + b*y        │
│ c  d │   │ y │   │ c*x + d*y        │
└      ┘   └   ┘   └                  ┘
```

但**记这个公式不如记一句话**：

> 矩阵的每一列，是"基向量变换后的新位置"。

### 基向量视角（3B1B 的神来之笔）

二维空间里，有两个最特殊的向量：
- `i = [1, 0]` — 指向 x 轴正方向
- `j = [0, 1]` — 指向 y 轴正方向

任何向量 `[x, y]` 都可以写成 `x*i + y*j`。

**矩阵 A 做的事**：
- 把 `i` 变到 A 的第一列
- 把 `j` 变到 A 的第二列
- 其他所有向量随之被"拖动"

```
        ┌     ┐
    A = │ 2 0 │   第一列 = i 的新位置 = [2, 0]
        │ 0 3 │   第二列 = j 的新位置 = [0, 3]
        └     ┘
```

这就是为什么矩阵只要两列就能描述整个平面的变换——基向量定了，一切都定了。

### 几种重要的变换

| 矩阵 | 效果 |
|------|------|
| `[[2,0],[0,2]]` | 整体放大 2 倍 |
| `[[1,0],[0,-1]]` | 关于 x 轴翻转 |
| `[[0,-1],[1,0]]` | 逆时针旋转 90° |
| `[[1,1],[0,1]]` | "剪切"变换（天花板被推歪） |

### 这有什么用

**神经网络的每一层本质上就是一次矩阵变换 + 非线性**：

```python
output = activation(W @ input + b)
```

`W` 是矩阵（空间变换），`+ b` 是平移，`activation` 是非线性扭曲。一层层叠加，就是把输入向量"揉"到能被最后一层分类的形状。

## §4 矩阵乘法：两次变换的叠加

### 直觉

两个矩阵相乘 `A @ B` = **先做 B 变换，再做 A 变换**。

注意顺序！从右往左读。

### 为什么矩阵乘法不满足交换律

`A @ B ≠ B @ A` 通常情况下。

想象：
- A = 旋转 90°
- B = 沿 x 轴拉伸 2 倍

"先旋转再拉伸" 和 "先拉伸再旋转" 显然结果不同。这就是矩阵乘法不可交换的几何原因。

### 计算公式

（能看几何就别算公式，需要时查即可）

```
(A @ B)[i, j] = A 的第 i 行 · B 的第 j 列（点积）
```

### 动手验证

`code/04_compose.py` 会把两次变换分步画出来，让你直观看到"叠加"的过程。

## §5 行列式：面积的放大倍数

### 直觉

2×2 矩阵的行列式 `det(A)` 回答一个问题：

> 单位正方形经过 A 变换后，面积变成了原来的几倍？

- `det = 2` → 面积变 2 倍
- `det = 1` → 面积不变（比如纯旋转）
- `det = 0` → 面积被压扁成 0（空间被降维了）
- `det = -1` → 面积不变但方向翻转（镜像）

### det = 0 的重要意义

**如果 det(A) = 0，意味着变换把二维空间压成了一条线（甚至一个点）**。

这是灾难性的——信息丢失了，没法"还原"（即矩阵不可逆）。

ML 里：如果你的数据矩阵行列式接近 0，模型训练会不稳定。这叫"矩阵病态"。

### 计算公式

2×2 情况：

```
    ┌     ┐
A = │ a b │     det(A) = ad - bc
    │ c d │
    └     ┘
```

3×3 以上的公式复杂，用 `np.linalg.det(A)` 让计算机算就行。

## §6 点积与范数

### 范数：向量的长度

对于向量 `v = [x, y]`：

**L2 范数（欧几里得长度）**：`||v||₂ = √(x² + y²)`  — 就是勾股定理

**L1 范数**：`||v||₁ = |x| + |y|`  — 走曼哈顿街区的距离

### 为什么 ML 里两种范数都常见

- L2 → 对应岭回归（Ridge 正则化），倾向于让权重**均匀变小**
- L1 → 对应 Lasso 正则化，倾向于让**一部分权重直接变成 0**（特征选择效果）

### 点积：两个向量有多"像"

`a · b = a1*b1 + a2*b2 + ... + an*bn`

但记住几何含义更重要：

```
a · b = ||a|| * ||b|| * cos(θ)
```

其中 θ 是两个向量的夹角。

- `a · b > 0` → 夹角小于 90°，**同向**
- `a · b = 0` → 夹角 90°，**垂直**（专业术语"正交"）
- `a · b < 0` → 夹角大于 90°，**反向**

### 为什么 ML 里到处是点积

因为点积衡量相似度。

- 推荐系统里"用户向量 · 商品向量"越大，越可能喜欢
- 神经网络里 `w · x + b` 就是"权重和输入的相似度加上偏置"
- 余弦相似度 = 归一化后的点积

## §7 特征值与特征向量

### 直觉

在一个矩阵变换下，**大多数向量方向会改变**（被旋转、被扭曲）。

但有一类特殊的向量，**它们方向不变，只是被拉长或缩短**。这些就是**特征向量**，拉长/缩短的倍数就是**特征值**。

### 公式

```
A @ v = λ * v
```

读作："矩阵 A 作用在 v 上，等价于把 v 乘一个常数 λ"。

- `v` 是特征向量
- `λ`（lambda）是特征值

### 几何图像

想象一块橡皮布被变换 A 拉扯。大多数点被拉歪了，但沿着某几条特定方向的点**只是沿着自己原来的方向被拉长或缩短**。这几条方向就是特征向量。

### 为什么 ML 里特征向量极其重要

**PCA（主成分分析）** 的核心：数据协方差矩阵的特征向量，就是"数据变化最大的方向"。

降维 = 只保留特征值最大的几个特征向量对应的方向，把数据投影上去。

谷歌的 **PageRank** 算法本质上也是算一个特殊矩阵的最大特征向量。

### 计算

人类几乎不手算，用 `np.linalg.eig(A)` 让 NumPy 算。

### 动手验证
